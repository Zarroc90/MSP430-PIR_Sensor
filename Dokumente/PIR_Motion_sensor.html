Summary

This is a battery operated device. It has 1 Z-Wave chip - ZM5101, 1 TI MSP chip, which is connected to the REED and the PIR to detect open/close and motion start/stop.
LIGHT, TAMPER are connected to the ZM5101 chip

Operating instructions

Buttons

There is Tamper button on the device. To push it, remove the case and use it as a regular button
Button events:

  • Single Click: send out sensor reports and Wake Up notification
  • Triple Click: Learn Mode
  • Hold 5 seconds, followed by Triple click: Device Reset locally

Z-Wave network management

Learn Mode

To enter Learn Mode use triple click on the button.

Reset

Hold Button for 5 seconds, the LED will start blinking. Afterwards triple click the button to reset the device

Wake up notification

To send wake up notification use single button click

Indicator LEDs

ZM5101 has 1 indicator LED for Learn Mode activity, reset, etc.

Wake Up Sources

Following sources can wake up the device:
1. User opens the case and releases Tamper -> INT1 goes LOW
2. MSP430 drives INT1 pin LOW to report Motion change
3. Wake up Timer

Light Sensor management

ZM5101 wakes up every 60 seconds, when included to a Z-Wave network. Each Wake up it measures new Luminescence value to see if it must be reported to the controller. The value can be reported by Configuration Param interval (param 5) or by hysteresis (param 6).

ADC light measurement

The light circuit consist of a photo-transistor, connected through resistor to ZM5101 ADC pin.
Real life measurements:

Conditions Voltage ADC Light Report
Handy Photo 0.07 5/255 98 %
Room light 0.2 17/255 94 %
Dark 2.8 238/255 7 %

Motion Sensor

Motion sensor is measured by MSP430. Once motion is detected, MSP430 wakes up ZM5101 and ZM5101 reports to the controller about the change. See workflow diagrams for more details

Reed Sensor

Reed sensor is also connected to the MSP430. If change is detected, MSP430 wakes up ZM5101 and ZM5101 reports to the controller about the change. See workflow diagrams for more details.

Tamper Sensor

Once the case is removed, the Tamper goes LOW, waking up the ZM5101. Once woken up, the ZM5101 doesn’t go to sleep until the case is back on place. During this stage, the Tamper can be used as a button to perform network activity with the device.

Warning
This will kill the battery if the device is awaken for too long. The device uses slow LED blinking to indicate it’s not sleeping

Association Groups

The Device has 3 association groups, max nodes 10:

  1. Lifeline: All of the device status reports are sent to this group:
    • BATTERY_REPORT
    • NOTIFICATION_REPORT (Tamper, Access, etc.)
    • DEVICE_RESET_LOCALLY
    • SENSOR_MULTILEVEL_REPORT
    • SENSOR_BINARY_REPORT
  2. Motion Basic Control: Once motion is detected, Basic Set On is sent to this group with value, set up by config parameter 1. Once Motion is Idle, Basic Set Off is sent with value, set up by config parameter 2. Commands are sent to this group, only if light level is below value, set in config param 7.
  3. Reed Basic Control: Once motion is detected, Basic Set On is sent to this group with value, set up by config parameter 3. Once Motion is Idle, Basic Set Off is sent with value, set up by config parameter 4.

Params:

№ Size Range Default Description
1 1 0 - 255 255 Basic Set Value, sent to group 2, when motion detected
2 1 0 - 255 0 Basic Set Value, sent to group 2, when motion turns idle
3 1 0 - 255 255 Basic Set Value, sent to group 3, when door opens
4 1 0 - 255 0 Basic Set Value, sent to group 3, when door closes
5 2 0 - 65535 3600 Periodic interval to send Luminescence value(0 - disabled)
6 1 0 - 50 10 Hysteresis value to automatically report Luminescence on change
7 1 0 - 100 0 Basic Set is sent to association group, when light is below this level. 0 - turned off
